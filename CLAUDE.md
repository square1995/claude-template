# 開発ルール

このファイルは、Claude Code が作業する際に必ず参照する共通ルールです。

---

## ユーザーについて

- プログラミング初心者であり、コードの詳細は分からない
- 必要に応じて、操作手順や用語の意味を簡潔に説明する

## 出力ルール

- 修正が必要な関数のみ「完全版」で提供する(部分的な抜粋は不可)
- 新規の関数を提案する場合は、ファイル内のどこに入れるかを明示する
- 処理の詳しい説明は、ユーザーから聞かれるまで不要
- コピペ用のコマンドや指示は、必ずコードブロック(```で囲む)で出力する

## デプロイ・Git 操作のルール

- main ブランチへのプッシュは、必ずユーザーの確認を取ってから実行する
- 強制プッシュ(force push)はユーザーの確認なしに絶対に行わない
- ブランチの削除はユーザーの確認なしに行わない
- コミットメッセージは日本語で簡潔に書く

## NEVER(過去のトラブルから学んだ禁止事項)

- ユーザーの確認なしに本番環境(main ブランチ)を変更しない
- 動作未確認のコードをデプロイしない
- 既存の機能を予告なく削除・改変しない
- 同じエラーで2回以上ループした場合、勝手に試行錯誤せずユーザーに状況を報告する

## トラブル時の対応

- エラーが出た場合は、勝手に推測せず、エラーメッセージをそのまま報告する
- 修正案が複数考えられる場合は、選択肢を提示してユーザーに選んでもらう
